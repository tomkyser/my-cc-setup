## TRACK A — Detailed Instructions for Each Concept

**For EACH of the 7 concepts, produce these 4 sections:**

### Steel-Man Case
Present the strongest possible argument FOR the concept. Draw from Synthesis v2's own reasoning. Be genuinely charitable — this is the concept's best day in court.

### Stress-Test
Apply adversarial pressure using:
- Research findings from 260318-x21-RESEARCH.md (empirical evidence)
- Mechanistic constraints from Synthesis v2 Section 1 (no cognition, only interpolation; no extrapolation; no grounding)
- Practical constraints (Dynamo's zero-dependency CJS architecture, hook timing budgets, cost projections)
- The existing INNER-VOICE-SPEC.md mechanical design (Section 4) — does the concept integrate or conflict?
- Cross-reference with LEDGER-CORTEX-ANALYSIS.md verdicts where applicable

### Verdict
One of: GO / CONDITIONAL GO / DEFER / NO-GO
- GO: Concept is sound and should be integrated as specified
- CONDITIONAL GO: Sound with modifications; specify exact conditions
- DEFER: Concept may have value but should not be in v1.3; specify when and why
- NO-GO: Concept fails under scrutiny; specify what replaces it (if anything)

Flag confidence level: HIGH / MEDIUM / LOW on every verdict.

### Integration Recommendation
For GO and CONDITIONAL GO:
- Reference specific INNER-VOICE-SPEC.md section numbers that need modification
- Reference specific MASTER-ROADMAP requirement IDs (CORTEX-XX) affected
- Specify v1.3 vs v1.4 boundary

For DEFER:
- Specify what milestone and what conditions would promote it

For NO-GO:
- Specify what (if anything) replaces it and where in the spec

## CRITICAL RESEARCH FINDINGS — These MUST be incorporated:

1. **Concept 7 (Subagents):** Hooks CANNOT spawn subagents natively. `agent` hooks are verification-only (yes/no output). `claude -p` has 5-15s cold start. Research verdict is NO-GO for hot path, GO for pure CJS with direct API calls. The steel-man must acknowledge this kills the Synthesis v2 Section 2 substrate assumption. The replacement is the pattern already used by curation.cjs — direct HTTP API calls from CJS modules.

2. **Concept 3 (Variable substitution):** Research finds this is cosmetic debiasing — transformer attention activates from semantic content, not labels. The steel-man must present the strongest case but the stress-test must be honest about the mechanism. Research suggests adversarial counter-prompting as a replacement.

3. **Concept 1 (Frame-first pipeline):** Domain classification at <100ms is achievable via keyword/regex (v1.3) or embedding similarity (v1.4). This is a CONDITIONAL GO from research — the pipeline order change is sound but domain classification must start simple.

4. **Concept 5 (REM consolidation):** Well-validated by literature (Hindsight, A-MEM, rem-sleep skill, academic papers). Stop hook synthesis is a GO. Inter-session consolidation is correctly deferred to v1.4.

5. **Concept 6 (Scalar compute):** Fan-out at 500 entities / 1500 relationships / 3 frames / 4 dimensions / 2 hops = ~960 raw chains, prunable to ~32 LLM calls (4-6 batched). Hot path: 0 LLM calls (deterministic scoring). Deliberation: 4-6 batched calls. v1.3 must be 1-hop only.

6. **Concept 4 (IV memory):** No direct research finding — assess based on architectural analysis. Consider: graph density requirements, storage growth, relationship to existing inner-voice-state.json schema.

7. **Concept 2 (User-relative definitions):** Research finds feasible but shallow on sparse graphs (<200 entities). Defer explicit definitions to v1.4; v1.3 integrates graph context directly.
