<objective>
Fix versioning to support milestone pre-release scheme with -M{N} suffixes.

Purpose: The project uses milestone releases (v1.3-M1, v1.3-M2) but the current version comparison logic only handles plain X.Y.Z semver. This causes update checks to break when VERSION contains a milestone suffix or when GitHub tags use the v1.3-M2 format.

Output: Updated VERSION file, milestone-aware compareVersions, normalizeVersion helper, and comprehensive tests.
</objective>
