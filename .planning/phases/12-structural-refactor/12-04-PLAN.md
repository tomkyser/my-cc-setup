```markdown
# ===================================================
# Memory System: Dynamo Knowledge Graph CLI
# ===================================================

## Dynamo CLI for Memory Operations
- All memory operations go through the `dynamo` CLI at `~/.claude/dynamo/dynamo.cjs`
- Run commands via: `node ~/.claude/dynamo/dynamo.cjs <command> [options]`
- The global toggle controls all memory access -- if Dynamo is disabled, all commands error

## Available Commands
| Command | Description | Use When |
|---------|-------------|----------|
| `dynamo search <query>` | Search knowledge graph for facts and entities | Looking up what you know about a topic, recalling decisions |
| `dynamo search <query> --facts` | Search for facts (relationships) only | When you need specific relationship data |
| `dynamo search <query> --nodes` | Search for entity nodes only | Finding specific entities, people, tools, concepts |
| `dynamo remember <content>` | Store a memory in the knowledge graph | User says "remember this", storing decisions/patterns/prefs |
| `dynamo recall` | Retrieve recent episodes | Reviewing what was stored recently |
| `dynamo edge <uuid>` | Inspect a specific relationship | Getting details about a particular fact |
| `dynamo forget <uuid>` | Delete an episode | User says "forget this", removing incorrect info |
| `dynamo forget --edge <uuid>` | Delete a specific relationship | Removing a specific incorrect fact |
| `dynamo clear --scope <scope> --confirm` | Clear all data for a scope | DESTRUCTIVE -- only on explicit user request |
| `dynamo health-check` | Check server and database health | Diagnosing connection issues |
| `dynamo status` | Show enabled/disabled state | Checking if Dynamo is active |
| `dynamo toggle on\|off` | Enable/disable Dynamo globally | Maintenance or troubleshooting |

## Command Options
- `--scope <scope>`: Memory scope (default: global). Values: `global`, `project-<name>`, `session-<ts>`, `task-<desc>`
- `--format json`: Structured JSON output for programmatic processing
- `--format raw`: Full source content from graph for deep context
- Default: Human-readable text output

## How to Choose Format
- Use default (no flag) for conversational responses to the user
- Use `--format json` when you need to parse the response programmatically
- Use `--format raw` when you need the complete, unfiltered graph data

## Scoping
| Scope | Format | Contents |
|-------|--------|----------|
| Global | `global` | User prefs, workflow patterns, coding style |
| Project | `project-<name>` | Architecture, decisions, conventions |
| Session | `session-<timestamp>` | Conversation summaries |
| Task | `task-<descriptor>` | Task requirements, progress |

**Note:** Scope values must contain only alphanumeric characters, dashes, or underscores (no colons).

## CRITICAL: Flat-File Memory is DISABLED
- Do NOT use the auto memory directory (~/.claude/projects/*/memory/)
- Do NOT create or update MEMORY.md files
- ALL memory operations go through Dynamo CLI or are handled by hooks

## How Memory Works (Automatic -- Zero User Intervention)
Hooks inject context automatically at key lifecycle points:
- **SessionStart**: Global preferences + project context + recent sessions
- **UserPromptSubmit**: Semantic search augments every prompt with relevant memories
- **PostToolUse (Write/Edit)**: File changes captured to knowledge graph
- **PreCompact**: Key knowledge preserved before context window compression
- **Stop**: Session summarized and stored

## Using Injected Context
When you see `[GRAPHITI MEMORY CONTEXT]` or `[RELEVANT MEMORY]`:
- This is curated context from the knowledge graph -- use it
- Don't re-query for information already injected
- If context seems stale or insufficient, run `dynamo search` directly

## Explicit Memory Operations
- **"Remember this"**: `dynamo remember "<content>" --scope <scope>`
- **"What do you know about X"**: `dynamo search "<query>" --scope <scope>`
- **"Forget this"**: `dynamo forget <uuid>` (get UUID from search results)
- **Architectural decisions**: Proactively store via `dynamo remember` in project scope
- **Discovered patterns**: Store in project scope

## Graphiti Infrastructure
- Docker stack: `~/.claude/graphiti/docker-compose.yml` (started separately)
- Start: `dynamo start` or `~/.claude/graphiti/start-graphiti.sh`
- Stop: `dynamo stop` or `~/.claude/graphiti/stop-graphiti.sh`
- Health: `dynamo health-check`
- Neo4j browser: `http://localhost:7475`
```
