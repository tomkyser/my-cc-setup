**Part B: Add `generate-session-name` subcommand to graphiti-helper.py**

Add a new function `generate_session_name(text: str) -> str`:
```python
def generate_session_name(text: str) -> str:
    """Generate a 3-5 word session name via Haiku."""
    if not OPENROUTER_API_KEY:
        # Fallback: first 5 words of text, title-cased
        words = text.split()[:5]
        return " ".join(words).title()[:50] if words else ""

    prompt_config = PROMPTS.get("generate_session_name", {})
    system_prompt = prompt_config.get("system", "Generate a concise 3-5 word session name. Output ONLY the name.")
    user_template = prompt_config.get("user", "SESSION CONTEXT:\n{context}\n\nGenerate a single 3-5 word session name.")

    user_prompt = user_template.format(context=text[:2000])

    try:
        resp = httpx.post(
            CURATION_API_URL,
            json={
                "model": CURATION_MODEL,
                "messages": [
                    {"role": "system", "content": system_prompt},
                    {"role": "user", "content": user_prompt}
                ],
                "max_tokens": 30,
                "temperature": 0.3
            },
            headers={
                "Authorization": f"Bearer {OPENROUTER_API_KEY}",
                "Content-Type": "application/json"
            },
            timeout=10.0
        )
        if resp.status_code == 200:
            data = resp.json()
            name = data["choices"][0]["message"]["content"].strip()
            # Sanitize: remove quotes, limit length
            name = name.strip('"\'').strip()
            return name[:80] if name else ""
    except Exception:
        pass

    # Fallback on any failure
    words = text.split()[:5]
    return " ".join(words).title()[:50] if words else ""
```

**Part C: Add preliminary naming to prompt-augment.sh**

Add a "first prompt" detection mechanism using a PPID-keyed flag file (same pattern as health check). Add AFTER the health check block and BEFORE the memory search block:

```bash
# --- Preliminary session naming (first substantial prompt only) ---
SESSION_NAMED_FLAG="/tmp/graphiti-session-named-${PPID:-$$}"
if [ ! -f "$SESSION_NAMED_FLAG" ]; then
  touch "$SESSION_NAMED_FLAG"
  # Generate preliminary session name from first prompt
  TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
  SESSION_NAME=$($HELPER generate-session-name --text "$PROMPT" 2>/dev/null || echo "")
  if [ -n "$SESSION_NAME" ]; then
    $HELPER index-session \
      --timestamp "$TIMESTAMP" \
      --project "$PROJECT" \
      --label "$SESSION_NAME" \
      --labeled-by "auto" 2>/dev/null || true
  fi
fi
```

This requires PROJECT to be set before the naming block. The current prompt-augment.sh already detects project BEFORE memory search, so the naming block goes between project detection and memory search.

Ensure the naming block uses `2>/dev/null || true` on the index-session call to avoid breaking the prompt augmentation flow if naming fails. The flag file prevents naming on subsequent prompts.

**Part D: Update session-summary.sh to include refined name**

In session-summary.sh, REPLACE the existing `index-session` call (which has `--label ""`) with a version that generates a refined name from the summary:

```bash
# Generate refined session name from summary
SESSION_NAME=""
if [ -n "$SUMMARY" ]; then
  SESSION_NAME=$($HELPER generate-session-name --text "$SUMMARY" 2>/dev/null || echo "")
fi

# Write session entry to local index with refined name
if ! $HELPER index-session \
    --timestamp "$TIMESTAMP" \
    --project "$PROJECT" \
    --label "$SESSION_NAME" \
    --labeled-by "auto"; then
  log_error "Failed to write session index entry"
fi
```

This overwrites the preliminary name from prompt-augment.sh with a refined name based on the full session summary. The `labeled_by=auto` ensures that user-assigned labels (labeled_by=user) are preserved by index-session's existing guard.

If SUMMARY is empty, SESSION_NAME will be empty, and index-session will be called with an empty label. This is intentional -- the preliminary name from prompt-augment.sh is already in sessions.json and will be preserved if index-session sees an empty label replacing an existing non-empty one. Update `cmd_index_session` to NOT overwrite with empty label when existing label is non-empty:

Add to `cmd_index_session` in graphiti-helper.py, inside the `if existing:` block, after the `labeled_by == "user"` check:
```python
if existing:
    if existing.get("labeled_by") == "user":
        return
    # Don't overwrite non-empty label with empty one
    if not label and existing.get("label"):
        return
    existing["label"] = label
    existing["labeled_by"] = labeled_by

        existing["project"] = project
```
