**File 1: `~/.claude/graphiti/health-check.py`**

A streamlined health check script (not a verbose diagnostic — this is a quick status tool). Uses the same Python venv as graphiti-helper.py.

**Checks (stage-by-stage, in order):**

1. **check_docker**: Docker containers `graphiti-neo4j` and `graphiti-mcp` running.
   - Command: `docker ps --filter name=graphiti --format '{{.Names}} {{.Status}}'`
   - Output: `[OK] Docker: graphiti-neo4j (Up), graphiti-mcp (Up)` or `[FAIL] Docker: {which container is missing/unhealthy}`

2. **check_neo4j**: Neo4j HTTP reachable on localhost:7475.
   - HTTP GET `http://localhost:7475`
   - Output: `[OK] Neo4j: HTTP reachable on port 7475` or `[FAIL] Neo4j: {error}`

3. **check_graphiti_api**: Graphiti health endpoint at localhost:8100/health.
   - HTTP GET `http://localhost:8100/health`
   - Output: `[OK] Graphiti API: healthy` or `[FAIL] Graphiti API: {error}`

4. **check_mcp_session**: MCP session initialization succeeds.
   - POST initialize JSON-RPC to `http://localhost:8100/mcp`
   - Output: `[OK] MCP Session: initialized (session_id: {first 8 chars}...)` or `[FAIL] MCP Session: {error}`

5. **check_env_vars**: Required environment variables are set (load from `~/.claude/graphiti/.env`).
   - Check `OPENROUTER_API_KEY` and `NEO4J_PASSWORD` are non-empty
   - Output: `[OK] Environment: OPENROUTER_API_KEY set, NEO4J_PASSWORD set` or `[FAIL] Environment: {which var is missing}`
   - IMPORTANT: Never print actual key values — only confirm they are non-empty

6. **check_canary_roundtrip**: Write a canary episode, then read it back — the definitive proof of end-to-end data flow.
   - Generate a unique canary ID: `diag-canary-{uuid4_short}` (first 8 chars of UUID)
   - Write: call `add_memory` MCP tool with `episode_body: "Health check canary: {canary_id} at {ISO_TIMESTAMP}"`, `group_id: "global"`, `source: "text"`, `source_description: "health-check"`
   - Wait 2 seconds (allow Graphiti to process — entity extraction is async)
   - Read: call `search_memory_facts` with `query: "health check canary {canary_id}"`, `group_ids: ["global"]`, `max_facts: 3`
   - Output: `[OK] Canary round-trip: write succeeded, read found canary {canary_id}` or partial results like `[WARN] Canary round-trip: write succeeded but read returned empty (may need more time for indexing)` or `[FAIL] Canary round-trip: {which step failed and why}`
   - Cleanup: optionally delete the canary episode (Claude's discretion — a `--cleanup` flag is fine, or just leave it as a harmless timestamp marker)

**Output format:**
```
=== Graphiti Memory Pipeline Health Check ===
Timestamp: {ISO_TIMESTAMP}

[OK]   Docker: graphiti-neo4j (Up), graphiti-mcp (Up)
[OK]   Neo4j: HTTP reachable on port 7475
[OK]   Graphiti API: healthy
[OK]   MCP Session: initialized (session_id: abc12345...)
[OK]   Environment: OPENROUTER_API_KEY set, NEO4J_PASSWORD set
[OK]   Canary round-trip: write succeeded, read found canary diag-canary-f47ac10b

Result: 6/6 checks passed — pipeline healthy
```

Or on failure:
```
=== Graphiti Memory Pipeline Health Check ===
Timestamp: {ISO_TIMESTAMP}

[OK]   Docker: graphiti-neo4j (Up), graphiti-mcp (Up)
[OK]   Neo4j: HTTP reachable on port 7475
[OK]   Graphiti API: healthy
[FAIL] MCP Session: httpx.ConnectError — connection refused
[SKIP] Environment: (skipped — earlier stage failed)
[SKIP] Canary round-trip: (skipped — MCP session failed)

Result: 3/6 checks passed — pipeline UNHEALTHY
First failure: MCP Session
```

**Behavior notes:**
- If a stage fails, skip dependent stages (Docker must pass for Neo4j to be tested; MCP session must pass for canary)
- Exit code: 0 if all checks pass, 1 if any check fails
- Script accepts `--json` flag to output results as JSON (for programmatic consumption by future tools)
- Script accepts `--verbose` flag to include full response bodies (for debugging)
- Use `httpx` for HTTP calls, `subprocess` for Docker commands
- Shebang: `#!/usr/bin/env python3`
