**Part A: Add `cmd_verify_memory` to graphiti-helper.py**

Add a new `cmd_verify_memory(keep: bool = False)` function and argparse registration. The function performs a quick pass/fail check of the full memory pipeline (~30 seconds). Implementation:

1. Print header: `=== MEMORY SYSTEM VERIFICATION ===`
2. Define a checks list to collect results as `(name, status, detail)` tuples where status is "PASS" or "FAIL".
3. **Check 1 — Server Health**: Call `httpx.get(HEALTH_URL, timeout=5)`. PASS if status 200. FAIL with connection error message otherwise.
4. **Check 2 — Global Scope Write**: Use MCPClient to call `add_memory` with `group_id="global"`, `text=f"[verify-memory] canary probe {uuid4()}"`, `source="verification"`. PASS if response has `result` key. FAIL with error text.
5. **Check 3 — Global Scope Read**: Sleep 2 seconds for indexing. Use MCPClient to call `search_memory_facts` with `group_id="global"`, `query="verify-memory canary probe"`. PASS if at least 1 result returned. FAIL if empty.
6. **Check 4 — Session Index**: Call `_load_sessions()`. PASS if list is non-empty (sessions exist). FAIL if empty with message "No sessions indexed — run backfill-sessions or start a new session".
7. **Check 5 — List Sessions**: Run `subprocess.run([sys.executable, __file__, "list-sessions", "--json", "--all"], capture_output=True, text=True, timeout=10)`. PASS if exit code 0 and stdout parses as non-empty JSON array. FAIL with stderr.
8. **Check 6 — View Session**: If Check 5 passed, extract first session's timestamp from the JSON output. Run `subprocess.run([sys.executable, __file__, "view-session", "--timestamp", timestamp, "--json"], capture_output=True, text=True, timeout=10)`. PASS if exit code 0 and stdout is non-empty. FAIL with stderr.
9. **Cleanup** (unless `--keep` flag): If canary was written in Check 2, note in output "Canary data will be cleaned by Graphiti's entity resolution". (Graphiti has no explicit delete-episode API exposed via MCP, so canary data is left to natural cleanup. Document this in the output.)
10. Print results table: each check on one line with `[PASS]` or `[FAIL]` prefix.
11. Print final summary line: `PASS: Memory system healthy (N/N checks passed)` or `FAIL: N issues found (X/Y checks passed)`.
12. Exit with code 0 if all passed, 1 if any failed.

Register in argparse:
```python
vm = subparsers.add_parser("verify-memory", help="Quick pass/fail verification of the full memory pipeline")
vm.add_argument("--keep", action="store_true", help="Keep canary test data (default: cleaned up)")
```

Add to the command dispatch block:
```python
elif args.command == "verify-memory":
    cmd_verify_memory(keep=args.keep)
```

Update the module docstring at the top of the file to include:
