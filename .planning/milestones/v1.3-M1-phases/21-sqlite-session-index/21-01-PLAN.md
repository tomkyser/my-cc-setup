    Functional API module (not class -- per research recommendation and codebase convention). Key design:

    ```javascript
    'use strict';
    const fs = require('fs');
    const path = require('path');
    const os = require('os');
    const resolve = require('../../lib/resolve.cjs');
    const { logError } = require(resolve('lib', 'core.cjs'));

    const DEFAULT_DB_PATH = path.join(os.homedir(), '.claude', 'graphiti', 'sessions.db');
    ```

    **Availability detection:**
    - Module-level `_sqliteAvailable = null` (lazy, checked once)
    - `isAvailable()`: try `require('node:sqlite')`, set `_sqliteAvailable = true` on success, `false` on catch. Log once via `logError('session-store', 'node:sqlite not available...')` on failure.

    **Connection management:**
    - `_connections = new Map()` keyed by absolute dbPath
    - `getDb(dbPath)`: if Map has open connection for dbPath, return it. Otherwise create `new DatabaseSync(dbPath)`, run `PRAGMA journal_mode=WAL`, run `PRAGMA synchronous=NORMAL`, run `CREATE TABLE IF NOT EXISTS sessions (timestamp TEXT PRIMARY KEY, project TEXT, label TEXT, labeled_by TEXT, named_phase TEXT)`, run `CREATE INDEX IF NOT EXISTS idx_sessions_project ON sessions(project)`, store in Map, return.
    - `closeDb(dbPath)`: get from Map, if open call `.close()`, delete from Map.
    - `fs.mkdirSync(path.dirname(dbPath), { recursive: true })` before creating DatabaseSync instance.

    **CRUD operations (all accept options with dbPath override):**
    - `upsertSession(timestamp, project, label, labeledBy, namedPhase, options)`:
      - `db.prepare('INSERT OR REPLACE INTO sessions (timestamp, project, label, labeled_by, named_phase) VALUES (?, ?, ?, ?, ?)').run(timestamp, project || '', label || '', labeledBy || '', namedPhase || null)`
      - Note: Uses INSERT OR REPLACE for upsert semantics. Migration uses INSERT OR IGNORE (different intent).
    - `getSession(timestamp, options)`:
      - `const row = db.prepare('SELECT * FROM sessions WHERE timestamp = ?').get(timestamp)`
      - Return `row ? { ...row } : null` (spread to convert null-prototype object, normalize undefined to null)
    - `getAllSessions(options)`:
      - Build query dynamically: base `SELECT * FROM sessions`, add `WHERE project = ?` if `options.project`, add `ORDER BY timestamp DESC`, add `LIMIT ?` if `options.limit`.
      - Use `db.prepare(query).all(...params)` and spread each row: `rows.map(r => ({ ...r }))`
    - `deleteSession(timestamp, options)`:
      - `const result = db.prepare('DELETE FROM sessions WHERE timestamp = ?').run(timestamp)`
      - Return `result.changes > 0`

    **Migration:**
    - `migrateFromJson(jsonPath, dbPath, options)`:
      - Check `isAvailable()` first, return `{ migrated: 0, skipped: 0, status: 'no-sqlite' }` if false
      - Try `fs.readFileSync(jsonPath, 'utf8')` + `JSON.parse()` -- handle ENOENT as `status: 'no-file'`, parse errors as `status: 'parse-error'`
      - Check `Array.isArray(sessions)` -- return `status: 'invalid-format'` if not
      - Prepare `INSERT OR IGNORE INTO sessions (timestamp, project, label, labeled_by, named_phase) VALUES (?, ?, ?, ?, ?)`
      - Wrap in `db.exec('BEGIN')` / `db.exec('COMMIT')` with try/catch for `db.exec('ROLLBACK')`
      - For each entry: if `!entry || typeof entry.timestamp !== 'string'`, increment skipped, `logError('session-migration', 'Skipping invalid entry: ' + JSON.stringify(entry))`, continue
      - Otherwise `ins.run(entry.timestamp, entry.project || '', entry.label || '', entry.labeled_by || '', entry.named_phase || null)`, increment migrated
      - Return `{ migrated, skipped, status: 'ok' }`
