**2. Add Troubleshooting section** -- Insert BEFORE the jailbreak footer, AFTER "Graphiti Infrastructure":

```
## Troubleshooting

### "Dynamo is disabled" error
- Check status: `dynamo status`
- Re-enable: `dynamo toggle on`
- For development with Dynamo off globally: set `DYNAMO_DEV=1` in your environment

### Stack not starting
- Run `dynamo health-check` to identify which stage fails
- Check Docker is running: `docker ps`
- Check container status: `docker compose -f ~/.claude/graphiti/docker-compose.yml ps`
- View logs: `docker logs graphiti-mcp --tail 50`
- Full restart: `dynamo stop && dynamo start`

### Stale session IDs
- Restart Claude Code entirely -- MCP client caches session IDs

### Hook errors
- Check error log: `cat ~/.claude/dynamo/hook-errors.log`
- Log auto-rotates at 1MB
- Hooks always exit 0 (never block Claude Code) -- errors are logged, not thrown

### Health check failures
- Run deep diagnostics: `dynamo diagnose`
- This runs 13 stages covering Docker, Neo4j, API, MCP, env vars, and more
- Each stage reports PASS/FAIL/SKIP with details

### Memory not persisting
- Verify pipeline: `dynamo verify-memory`
- This runs 6 checks: write, read, scope isolation, sessions, and round-trip
```
